📘 UI/UX DESIGN SPECIFICATION & GOOGLE STITCH PROMPT BUILDER: ASK-DC PLATFORMProject Focus: ASK-DC AIOps Platform (PT Astra Honda Motor & PT Semesta Teknologi Informatika)Methodology Foundation: Contextual Design (Collecting Data, Interpretation, Consolidation, Visioning, Storyboarding, UED, Prototyping)Ecosystem Integration: SmartPM (Integrated Preventive Maintenance Ecosystem)1. Executive Summary & Architectural AlignmentDokumen ini mendefinisikan arsitektur UI/UX komprehensif untuk ASK-DC Platform, sebuah sistem Artificial Intelligence for IT Operations (AIOps) sentral yang dirancang untuk mengotomatisasi pemantauan, deteksi insiden (Zabbix Integration), pemetaan aset fisik (DCIM Integration), dan analisis akar penyebab masalah secara otonom (Autonomous Root Cause Analysis Engine).Proyek SmartPM bertindak sebagai ekosistem pendukung terintegrasi. ASK-DC menarik data jadwal, riwayat pemeliharaan pencegahan (Preventive Maintenance), dan interaksi bot AI untuk memperkaya konteks triase insiden tanpa menduplikasi antarmuka internal SmartPM itu sendiri.Sesuai dengan revisi ruang lingkup terbaru, seluruh komponen Telegram Bot telah dihapus dari arsitektur sistem. Semua mekanisme notifikasi, penanganan perintah, dan interaksi agen cerdas dialihkan ke dalam antarmuka web terpadu: Web Dashboard Module dan Web Chat Box Module menggunakan mekanisme Internal Web Notification Service.2. Contextual Design Framework MappingSpesifikasi antarmuka yang dibangun dalam dokumen ini didasarkan secara ketat pada tahapan metode Contextual Design untuk merefleksikan kebutuhan nyata tim Data Center Operations:Collecting Data: Hasil observasi langsung dan one-to-one interview terhadap operator NOC menunjukkan adanya fragmentasi informasi. Saat terjadi kegagalan sistem (system outage), operator harus membuka tiga aplikasi berbeda (Zabbix untuk alert, spreadsheet manual untuk jadwal PM, dan aplikasi inventory terpisah).Interpretation (The 5 Work Models):Flow Model: Menggambarkan koordinasi antara Tim Monitoring, Tim Infrastruktur Jaringan, dan Vendor Eksternal saat alert dipicu.Sequence Model: Menguraikan langkah penanganan insiden: Alert Masuk $\rightarrow$ Triase Manual $\rightarrow$ Pencarian SOP $\rightarrow$ Kontak Vendor. ASK-DC memotong urutan ini melalui Autonomous RCA Engine.Cultural Model: Menunjukkan tingginya tekanan pemenuhan target Mean Time to Resolution (MTTR) dan urgensi menghindari denda SLA (Service Level Agreement).Artifact Model: Mengidentifikasi ketergantungan pada dokumen cetak SOP operasional dan log insiden historis berbentuk teks mentah.Physical Model: Menampilkan denah ruang kontrol NOC dengan batasan resolusi layar monitor pemantauan sentral.Data Consolidation: Mengintegrasikan model di atas ke dalam Affinity Diagram, menghasilkan kebutuhan kritis akan satu ruang kerja digital tunggal (Single Source of Truth) yang mampu melakukan injeksi konteks secara otomatis.Visioning: Transformasi proses operasional di mana AI Orchestration secara proaktif mengklasifikasikan intensi pengguna, memanggil pustaka alat (Tool Router), dan menyusun ringkasan bahasa alami dari data agregat kegagalan infrastruktur.Storyboarding: Skenario interaksi pengguna dari deteksi kegagalan kapasitas komputasi hingga penayangan rekomendasi perbaikan berbasis Knowledge Base.User Environment Design (UED): Pemetaan Focus Areas (Workspace Utama, Area Analisis RCA, Konsol Konfigurasi Integrasi, Pusat Pengetahuan Operasional).Prototyping: Pengujian purwarupa struktural sebelum kode diimplementasikan ke dalam Google Stitch.3. Comprehensive Google Stitch Prompt PacksBerikut adalah paket perintah (Prompt Packs) terstruktur yang dirancang khusus untuk dimasukkan ke dalam Google Stitch Engine guna menghasilkan rancangan antarmuka, tata letak komponen, dan cetak biru visual ASK-DC.PROMPT PACK 1: Shell Architecture & Dashboard Layout FoundationTarget Stitch Component: Base Layout Shell, Navigation Sidebar, Topbar Area, User Context Provider.Contextual Design Reference: Flow Model (Aturan koordinasi tim) & Physical Model (Keterbatasan visual workstation NOC).Stitch Prompt Code:PlaintextCreate a highly professional, enterprise-grade Web Dashboard Shell for an AIOps application named 'ASK-DC'. 
The design system must follow a modern, clean 'Metro UI' or dark-themed professional engineering layout tailored for 24/7 Network Operations Center (NOC) environments.

Layout Constraints & Specifications:
1. Master Container: Use standard layout structures (no raw Markdown blocks, no display flex/grid on body level). Implement static/block positioning with auto margins and structured layouts for maximum rendering consistency.
2. Sidebar Navigation: Located on the left side, fixed width. It must contain the following links with desaturated, high-contrast icons:
   - Dashboard Overview
   - Problem Workspace (Active & History)
   - Host Infrastructure Directory
   - Knowledge Base Management
   - Integration & System Admin Configuration
3. Topbar Area: Clean horizontal bar containing:
   - System Health Status Indicator (Dynamic connection lights for Zabbix, DCIM, and SmartPM ecosystems).
   - Internal Notification Center Icon (Bell icon displaying real-time alert counters, replacing the deprecated Telegram Bot Notification mechanism).
   - User Profile Dropdown (Logged-in user details, Role-Based Access Control indicator like 'Admin' or 'Operator').
4. Content Area: Fluid container with a desaturated off-white background (#f4f6f9) or deep dark charcoal (#121212) depending on theme selection. Ensure clear typography spacing using a line height of 1.25.
5. Cross-Project Visual Element: Include a dedicated section within the Host/Problem details link pointing directly to external SmartPM scheduling contexts without embedding full SmartPM sub-modules.
PROMPT PACK 2: The Incident & Problem Analytical WorkspaceTarget Stitch Component: Dashboard Overview Cards, Top 5 Problem Widget, Problem List Page, Problem Detail Screen, Host Context Panel.Contextual Design Reference: Sequence Model (Breakdown tahapan triase alert) & Artifact Model (Digitalisasi penanganan berkas insiden).Stitch Prompt Code:PlaintextGenerate an analytical user interface for the 'Problem & Incident Workspace' inside the ASK-DC platform. This view must synthesize real-time streaming data from Zabbix, asset locations from DCIM, and maintenance status from SmartPM.

Component Layout Requirements:
1. Summary Metrics Strip: Four distinct high-contrast card components placed side-by-side:
   - Total Problems (Cumulative counter with neutral color accent)
   - Active Problems (Critical status, bold desaturated red background highlight)
   - Resolved Incidents (Success status, muted green accent)
   - Mean Time to Resolution (MTTR) Tracker (Calculated performance metric text)
2. Top 5 Urgent Problems Table: A structured data grid displaying the top 5 current infrastructure issues. Columns must include: Severity level (color-coded badge), Problem Description, Affected Host, Trigger Time, and Action Button.
3. Problem Detail View: A dual-column split interface layout:
   - Left Column: Technical Evidence Collector snapshot (CPU/Memory utilization line charts, Zabbix event history data, and raw metric query listings).
   - Right Column: Autonomous RCA Engine Output Panel. This must render the AI-generated Root Cause Report, which contains 'Symptom Analysis', 'Impact Assessment', and 'Actionable Recommendation' based on internal Knowledge Base lookup.
4. Integrated Context Tool: On the Host detail subset, provide a dedicated UI border container showing physical environment context pulled from DCIM (Site, Room, Rack Position) and preventive context from SmartPM (Last PM Date, Next Scheduled Maintenance, Vendor Assigned).
PROMPT PACK 3: Visioning the AI Co-Pilot (Web Chat Box Module)Target Stitch Component: Floating Chat Widget, Conversation Stream UI, Context Injection Sidebar, Auto Prompt Trigger Bar.Contextual Design Reference: Visioning Stage (Automasi pemotongan birokrasi pencarian data) & Storyboarding (Skenario interaksi operator-AI).Stitch Prompt Code:PlaintextDesign an interactive Web Chat Box Module to act as the primary AI Operations Assistant within the ASK-DC interface. This module completely replaces all interactive chat tasks previously assigned to the Telegram Bot.

Interface and Interaction Pattern:
1. Floating Chat Trigger: A persistent, rounded button located at the bottom right corner of the dashboard screen. Clicking this toggles the view of the main chat container.
2. Conversation Container: A clean, vertical chat list panel. 
   - Message Bubbles: Left-aligned for AI responses, right-aligned for User inputs. Background fills must use soft, desaturated colors (e.g., light blue/gray for AI, neutral off-white for user) to ensure high readability.
   - Text Formatting: Complete support for bold text, lists, and structured code/log syntax snippets using standard monospace formatting with subtle interior container padding.
3. Context Injection Bar: A dynamic top banner inside the chat window that shows what dashboard page or server raw row data the user is currently looking at (e.g., 'Context Active: Server-DB-01 Problem Snapshot').
4. Auto Prompt Generator: Provide small clickable badge chips above the text input zone based on the active problem selected (e.g., 'Analyze RCA Pattern', 'Check SmartPM History', 'Find Matching SOP').
5. User Input Handler: A horizontal bottom input dock featuring a rich text area, an 'Enter' key shortcut listener, a file attachment button for documentation query, and a clear visual disabled state when the AI Engine is processing.
PROMPT PACK 4: User Environment Design (UED) Configuration MatrixTarget Stitch Component: Knowledge Base Administration Screen, Encryption Configuration Manager, Audit Log Viewer Panel.Contextual Design Reference: User Environment Design Diagram (Focus area mapping for data control).Stitch Prompt Code:PlaintextCreate the core administration layout representing the internal management Focus Areas of the ASK-DC User Environment Design matrix. This view provides data management, encryption parameters, and security overview.

Functional UI Areas to Build:
1. Knowledge Base Manager: 
   - Top Section: A clean drag-and-drop document upload interface supporting PDF, DOCX, and TXT files with a clear progress percentage indicator.
   - Main Grid: A data table listing uploaded operating procedures (SOPs). Columns: Document Title, Chunk Count, Embedding Status Badge, Last Indexed Timestamp, and Delete Action.
2. Integration & Secret Configuration Manager:
   - A multi-tab form layout separating configuration fields for: Zabbix API Endpoint, DCIM Database Client, SmartPM Webhook Credentials, and LLM Orchestration Provider Keys.
   - Password/Token Inputs: Masked text fields with an optional eye icon toggle for visibility. Add a prominent 'Test Connection' button adjacent to each credential segment that fires an active health check loop.
3. System Audit Trail Viewer:
   - A tabular layout displaying centralized system logs. Columns: Timestamp, User Account, Performed Action (e.g., 'Config Changed', 'KB Uploaded'), and Trace/Error ID mapping. Include a robust date-range picker and user filter dropdown menu.
PROMPT PACK 5: System Micro-Interactions, States, & Inter-System Error FailoversTarget Stitch Component: Empty State Panels, Shimmer Loading Filters, Global Error Toast Overlays, SmartPM Sync Disruption Alerts.Contextual Design Reference: Prototyping Stage (Validation of state transitions and edge cases).Stitch Prompt Code:PlaintextDevelop a comprehensive set of micro-interactions, layout variations, and structural states for the ASK-DC interface to validate usability during the contextual prototyping assessment phase.

States & Visual Behaviors Required:
1. Empty State UI: Designed for newly set up environments or cleared event logs. Render a centered, desaturated icon accompanied by clear typographic guidance text (e.g., 'No Active Problems Detected - System Healthy').
2. Shimmer/Skeleton Loading: Provide a skeleton loading overlay variant for the Top 5 Problem Widget and Problem List grid. Replace row content with animated, soft gradient blocks to simulate data fetching background jobs.
3. Integration Disruption Failover Warning: Design a specific inline alert banner that appears at the top of the Problem Detail Page if the external SmartPM API or DCIM mapping client times out. The banner must read: 'Partial Context Loaded: External SmartPM/DCIM Sync Timeout. Displaying Cached Evidence and Local RCA Analysis.' Use a soft, desaturated amber background with clear dark text.
4. API Validation Toast: Floating top-center alert toast notifications for operation feedback (Success: Muted green border; Failure: Muted red border).

UI/UX Consistency & Best Practices GuidelinesTo maintain an uncompromised standard of design and technical execution across all generated components, the frontend implementation must follow these rigid design laws:Typography & Contrast: Maintain a hierarchical scale where Primary Headings are set between 18-22pt, Section Headings at 13-15pt, and Body Text at 10-11pt. Line spacing must never drop below 1.15 to ensure operators can parse text during midnight shifts under dim lighting.Color Usage Rules: Use desaturated, dark, or muted primary headers with crisp white text. Reserve bright, saturated colors exclusively for status signals (e.g., Saturated Red for Critical Alert, Amber for Warning, Green for Active Operational Sync).Layout and Breaks Prevention: In WeasyPrint compilation or grid layout workflows, enforce page-break-inside: avoid strictly on small atomic building components such as single table rows, short chat bubbles, or localized metrics widgets. Never apply hard break rules on wide containing blocks to avoid empty layout gaps.Ecosystem Integrity: Ensure that the ASK-DC platform remains a cohesive standalone system. Data bindings connected to the SmartPM integration module must only be exposed inside contextual blocks (such as host histories or AI tools routers) to prevent interface cross-contamination.
